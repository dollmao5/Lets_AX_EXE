# Andrej Karpathy's AI & Coding Guidelines (카파시 가이드라인)

본 문서는 안드레 카파시(Andrej Karpathy)가 제시한 소프트웨어 엔지니어링, LLM(대형 언어 모델) 활용 및 AI 에이전트 개발에 관한 핵심 철학과 가이드라인을 정리한 규칙 정의서입니다.

---

## 💡 1. Core Philosophies (핵심 철학)

### 1) LLM을 새로운 OS(운영 체제)로 인식하라
* LLM은 단순히 다음 단어를 예측하는 텍스트 작성기가 아닙니다. LLM은 **자연어를 명령어 세트로 사용하는 완전히 새로운 연산 패러다임(Software 2.0 / LLM OS)**입니다.
* CPU가 파일을 읽고 쓰듯, LLM은 컨텍스트 윈도우(Context Window)를 램(RAM)으로 사용하고 도구(APIs, Web Search, Code Execution)를 활성화하여 연산을 수행합니다.

### 2) 프롬프팅은 새로운 '코딩'이다
* 프롬프트 작성은 컴파일러나 인터프리터에게 정교한 논리 흐름을 전달하는 프로그래밍 행위입니다.
* **명확성(Clarity)과 구체성(Specificity)**이 가장 핵심적인 문법입니다. 모호한 지시는 모호한 결과(Hallucination)를 낳습니다.

---

## 🛠️ 2. Development & Coding Principles (개발 및 코딩 원칙)

### 1) Start Simple, Iterate Fast (단순하게 시작하고, 빠르게 반복하라)
* 처음부터 복잡한 아키텍처나 AI 에이전트 흐름을 설계하지 마십시오.
* 작동하는 가장 작고 단순한 베이스라인(Baseline) 코드를 짠 다음, 모델의 동작을 관찰하며 점진적으로 기능을 고도화하십시오.

### 2) Guard against Hallucination (환각 대비 및 제어)
* 모델은 모르는 것도 아는 것처럼 대답하는 경향이 있습니다. 
* 이를 방지하기 위해 **"모르면 모른다고 솔직하게 답변할 것"** 또는 **"주어진 맥락(Context) 안에서만 근거를 찾아 답변할 것"**과 같은 명시적인 안전 제약조건(Guardrails)을 필수적으로 부여하십시오.

### 3) Chain of Thought (단계적 사고 유도)
* 복잡한 추론이나 코딩 작업을 요청할 때는 즉시 답을 도출하라고 요구하지 마십시오.
* **"문제를 단계별로 생각하라(Think step-by-step)"**는 명령을 통해 모델이 내부 논리 경로를 한 단계씩 풀어낸 뒤 최종 답변에 도달하도록 구조화하십시오.

### 4) Hands-on & Micro-testing (직접 검증 및 마이크로 테스트)
* AI가 작성한 코드를 맹신하지 마십시오. AI가 제안한 모듈이나 함수는 항상 로컬 격리 환경에서 가벼운 테스트(Micro-test)를 돌려 부작용이 없는지 직접 눈으로 검증해야 합니다.

---

## 🎯 3. Prompts & Agents Design Rules (프롬프트 및 에이전트 설계 규칙)

1. **역할 정의(System Role)**: 에이전트에게 항상 구체적이고 전문적인 페르소나(Persona)를 부여하십시오.
2. **출력 형식 강제(Structured Output)**: JSON, Markdown, XML 등 에이전트가 해석하기 쉬운 일관된 출력 템플릿 형식을 명확히 정의하십시오.
3. **Few-Shot Prompting**: 모호한 설명 백 줄보다 완벽하게 작성된 1~2개의 입출력 예시(Few-Shot)가 모델의 품질을 비약적으로 높입니다.

## 🚀 4. AX-CAMP Project Case Studies (프로젝트 실제 구현 사례)

본 프로젝트(`Lets_AX_EXE`)에서 위의 카파시 개발 가이드라인을 성공적으로 녹여낸 모범 실천 사례들을 수록합니다. 모든 에이전트 및 개발자는 신규 로직 추가 시 아래 패턴을 준수해야 합니다.

### 🛡️ 1) 방어적 예외 리다이렉트 설계 (Defensive Routing)
* **상황**: 렌더링 챕터 제외로 인해 발생한 숨겨진 해시 URL 직접 진입 예외 대응
* **실천**: 임의의 URL 접근 시 404 스크립트 에러를 무작정 던지지 않고, 시스템이 안전하게 동작 가능한 베이스라인 클립(`ch00-clip01`)으로 포워딩(우회 로직)하여 사용자 경험(UX)을 완벽히 보장하였습니다.

### 💾 2) 비파괴적 점진적 고도화 (Non-Destructive Iteration)
* **상황**: 특정 챕터(CH04, CH05) 노출 제외 요구 사항 반영
* **실천**: 원본 컨텐츠 리소스나 폴더를 물리적으로 삭제하는 무모한 방법 대신, 오직 백엔드와 프론트엔드의 **인메모리 렌더링 맵(visibleBlueprints, CLIENT_CATALOG_BLUEPRINTS)** 레이어만 주석 처리 및 조정하여 리스크를 0%로 만들고 쉬운 롤백성을 보장했습니다.

### 🔗 3) 런타임 방어 및 동적 DOM 제어 (Robust DOM Control)
* **상황**: 메인 화면 시간표 앵커 및 tr 노드 삭제/교정
* **실천**: 존재하지 않는 DOM 요소를 삭제하거나 텍스트를 읽을 때 발생할 수 있는 런타임 예외를 차단하기 위해 `Optional Chaining(?.)` 및 엄격한 조건 탐색을 결합하여 자바스크립트 스크립트 실행 무결성을 완벽하게 사수했습니다.
